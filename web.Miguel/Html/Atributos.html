  • Atributos globales.

    Los atributos globales en HTML son palabras especiales que se pueden utilizar con cualquier elemento HTML para mejorar su comportamiento y presentación. También se conocen como atributos estándares.

    Algunos de ellos son:

    1. <dir>

      Indica la direccionalidad del texto de los elementos. Puede tener los valores ltr (izquierda a derecha), rtl (derecha a izquierda) o auto.

    2. <title>

      Contiene texto que representa información relacionada con el elemento al que pertenece. Por ejemplo, en un vínculo, puede ser el título o la descripción del documento vinculado.

    3. <draggable>

      draggable Indica si el elemento se puede arrastrar mediante la API de arrastrar y soltar. Puede tener los valores true o false.

    4. <ID>

      El atributo id en HTML es un identificador único que se asigna a un elemento HTML para poder identificarlo en otras partes del código. Este atributo se puede utilizar para:

      1. Vincular elementos usando un identificador de fragmento

      2. Dar formato a la vista de operador con CSS

      3. Manipular el modelo de objeto de documento con código JavaScript y DHTML

      4. Modificar un elemento específico usando CSS o JavaScript

    5. <hidden>

      El hiddenatributo es un atributo booleano. Cuando está presente, especifica que un elemento aún no es relevante o ya no lo es.

      Los navegadores no deben mostrar elementos que tengan el hiddenatributo especificado. El atributo hidden también se puede utilizar para evitar que un usuario vea un elemento hasta que se cumpla alguna otra condición (como seleccionar una casilla de verificación, etc.). En ese caso, un código JavaScript podría eliminar el atributo oculto y hacer que el elemento sea visible.

  • Atributos form

    El atributo form en HTML especifica el formulario al que pertenece un elemento. El valor de este atributo debe ser igual al atributo id de un elemento <form> en el mismo documento.

    En el caso de los formularios, los atributos HTML son valores adicionales que configuran los elementos o ajustan su comportamiento:

    1. <accept>

      Indica los tipos de archivos que el servidor acepta

    2. <action>

      Indica el lugar donde se envían los datos del formulario. Su valor debe ser una dirección URL válida.

    3. <method>

      method: Se utiliza para cargar los datos. Los métodos más utilizados son GET y POST.

    4. <autocomplete>

      Indica si los controles del formulario pueden tener sus valores completados automáticamente por el navegador.

    5. <maxlenght>

      Limita el número máximo de caracteres que se pueden introducir en un campo.

    6. <size>

      Especifica la cantidad de espacio reservada para un campo.

    7. <value>

      Indica el valor inicial de un campo.

  • Atributos input

    1. <type>

      Define el tipo de input a usar, como texto, radio, fecha, etc.

    2. <name>

      Nombre del campo que se enviará en el formulario

    3. <value>

      Etiqueta input con valor por defecto para un campo

    4. <placeholder>

      Texto que se muestra en el campo hasta que el usuario empiece a escribir en él

    5. <Required>

      Especifica que se debe completar un campo de entrada antes de enviar el formulario

    6. <min>

      Define el valor mínimo que es aceptable y válido para el input

    7. <max>

      Define el valor máximo que es aceptable y válido para el input

    8. <multiple>

      Indica al selector de archivos que el usuario puede seleccionar varios archivos a la vez

    9. <accept>

      Indica los formatos de archivos que el usuario puede seleccionar
